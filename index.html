The problem

Generic converters treat ChemDraw exports as plain molecular files.

That strips the very context medicinal chemists need: plate labels, custom text, abbreviations, and drawing order.

🗂️

Stripped metadata

Plate context, compound IDs, and annotations are lost or duplicated across every row.

🧪

Lost chemistry

Abbreviations like OMe and Ph become placeholder atoms; R-groups are dropped or inconsistent.

🐌

Manual cleanup

You end up re-entering names, IDs, and properties into Excel by hand.

The solution

Built for SAR plates, compound libraries, and annotated screens.

Convertia preserves the chemistry as drawn and enriches it with RDKit descriptors and JPLogP lipophilicity.

🖱️

Drag-and-drop GUI

Drop · Browse · Convert

Drop SDF, CSV, CDX, or CDXML onto the window — output path and format are filled in automatically. Same portable Convertia.exe for GUI and CLI.

🏷️

Plate context preserved

CompoundID · Annotations · Title

Coordinate-based labels map to the nearest structure. IUPAC names link via ChemDraw chemicalproperty.

⚗️

Abbreviation expansion

OMe · Ph · Boc → real atoms

ChemDraw's embedded fragments are flattened into the parent structure, so formulas and SMILES are correct.

🔗

R-group handling

R1 · R2 → [*:1] · [*:2]

Generic nicknames become RDKit dummy atoms with atom-map numbers in canonical SMILES.

📊

Dual-namespace properties

ChemDraw_* vs RDKit columns

ChemDraw values are prefixed so they can never overwrite computed MolecularWeight, CLogP, or TPSA.

📋

Document order locked

XmlIndex 0..N−1

Every row carries a monotonic index matching the original drawing layout, even if downstream tools re-sort.

🧬

Batch descriptors

CLogP · MW · TPSA · HBD/HBA

Built-in RDKit/JPLogP computation: SMILES, formula, CLogP, TPSA, HBD, HBA, rotatable bonds, stereocenters.

Compared to generic tools

What generic converters miss, Convertia keeps.

Capability Generic tools Convertia
ChemDraw CDXML plates Often miss grouped structures, page text, and property links Native CDXML parser; walks <group> fragments in document order
Compound names (IUPAC captions) Usually lost Read via ChemDraw chemicalproperty / BasisObjects → Title column
Plate labels (IDs, notes) Not assigned per structure CompoundID and Annotations via spatial proximity
Abbreviations (OMe, Ph, Boc…) Often a single placeholder atom Expanded from ChemDraw's embedded fragment → correct formula & SMILES
R-groups (R1, R2…) Inconsistent or dropped Dummy atoms with map numbers ([*:1]) in SMILES
Row order on dense plates Undefined or re-sorted Monotonic XmlIndex (0..N−1) matches the drawing
ChemDraw vs computed properties Can overwrite each other Separate namespaces: ChemDraw_* vs RDKit columns
Batch descriptors Requires a separate workflow Built-in: SMILES, MW, formula, CLogP, TPSA, HBD/HBA, rotatable bonds, stereocenters

Output columns

Two namespaces, one clean row.

Properties are grouped by source so nothing is silently overwritten.

Parser metadata appears first in every row.

XmlIndex Title CompoundID Annotations

Standard RDKit descriptors, with CLogP computed by JPLogP (not Wildman–Crippen MolLogP).

SMILES MolecularWeight Formula CLogP TPSA NumHAcceptors NumHDonors NumRotatableBonds NumHeavyAtoms NumStereoCenters

ChemDraw properties keep the ChemDraw_ prefix. Empty label templates are dropped.

ChemDraw_LogP ChemDraw_MeltingPoint ChemDraw_MolecularWeight …any ChemDraw ChemProp

Command line

Same executable, terminal or desktop.

Double-click Convertia.exe for the GUI (drag-and-drop or file picker), or run it with arguments for batch conversion. CLI redirection works.

$ Convertia.exe input.cdxml -o output.csv
$ Convertia.exe library.sdf -o library.csv
$ Convertia.exe data.csv -o data.sdf --smiles-col SMILES
$ Convertia.exe table.cdxml -o table.sdf --3d
$ Convertia.exe huge.sdf -o huge.csv --no-properties
Known limitations

What Convertia does not do.

The tool extracts connectivity and data — it does not round-trip 2D drawings.

Critical
2D drawing elements (coordinates, bond styles, arrows, colors) are not preserved.
Critical
Complex stereochemistry may not survive roundtrip — verify after conversion.
Medium
Multi-attachment nicknames (linkers with 2+ attachment points) are not expanded.
High
Page text → structure assignment uses coordinates; dense plates may still mis-associate labels.
Critical
CDX binary requires OpenBabel; CDXML/SDF/CSV work without it.
High
CSV/SDF multiline fields may be fragile; use --strict to fail on raw newlines.
