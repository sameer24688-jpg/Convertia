The Problem

Generic converters treat ChemDraw exports as plain molecular files.

That strips the very context medicinal chemists need: plate labels, custom text, abbreviations, and drawing order.

🗂️

Stripped Metadata

Plate context, compound IDs, and annotations are lost or duplicated across every row.

🧪

Lost Chemistry

Abbreviations like OMe and Ph become placeholder atoms; R-groups are dropped or inconsistent.

🐌

Manual Cleanup

You end up re-entering names, IDs, and properties into Excel by hand.

The Solution

Built for SAR plates, compound libraries, and annotated screens.

Convertia preserves the chemistry as drawn and enriches it with RDKit descriptors and JPLogP lipophilicity.

🏷️

Plate Context Preserved

CompoundID · Annotations · Title

Coordinate-based labels map to the nearest structure. IUPAC names link via ChemDraw chemicalproperty.

⚗️

Abbreviation Expansion

OMe · Ph · Boc → real atoms

ChemDraw's embedded fragments are flattened into the parent structure, so formulas and SMILES are correct.

🔗

R-Group Handling

R1 · R2 → [*:1] · [*:2]

Generic nicknames become RDKit dummy atoms with atom-map numbers in canonical SMILES.

📊

Dual-Namespace Properties

ChemDraw_* vs RDKit columns

ChemDraw values are prefixed so they can never overwrite computed MolecularWeight, CLogP, or TPSA.

📋

Document Order Locked

XmlIndex 0..N−1

Every row carries a monotonic index matching the original drawing layout, even if downstream tools re-sort.

🧬

Batch Descriptors

CLogP · MW · TPSA · HBD/HBA

Built-in RDKit/JPLogP computation: SMILES, formula, CLogP, TPSA, HBD, HBA, rotatable bonds, stereocenters.

Command Line

Same executable, terminal or desktop.

Double-click Convertia.exe for the GUI, or run it with arguments for batch conversion. CLI redirection works.

$ Convertia.exe input.cdxml -o output.csv $ Convertia.exe library.sdf -o library.csv $ Convertia.exe data.csv -o data.sdf --smiles-col SMILES $ Convertia.exe table.cdxml -o table.sdf --3d $ Convertia.exe huge.sdf -o huge.csv --no-properties
Known Limitations

What Convertia does not do.

The tool extracts connectivity and data — it does not round-trip 2D drawings.

Critical
2D drawing elements (coordinates, bond styles, arrows, colors) are not preserved.
Critical
Complex stereochemistry may not survive roundtrip — verify after conversion.
Medium
Multi-attachment nicknames (linkers with 2+ attachment points) are not expanded.
High
Page text → structure assignment uses coordinates; dense plates may still mis-associate labels.
Critical
CDX binary requires OpenBabel; CDXML/SDF/CSV work without it.
High
CSV/SDF multiline fields may be fragile; use --strict to fail on raw newlines.
